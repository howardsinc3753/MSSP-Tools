Device Onboarding

Process for adding new FortiGate devices to the solution pack.

Current Status: Manual process. Automated tools planned for v1.2.0.

Manual Onboarding Steps

1
Generate API Token on FortiGate
System → Administrators → Create New → REST API Admin
2
Add to Credentials File
Edit config/fortigate_credentials.yaml
3
Update solution-pack.yaml
Add device to device_registry section
4
Test Connectivity
Run fortigate-health-check against new device

Credential File Format

config/fortigate_credentials.yaml
devices:
  new-device-id:
    host: "192.168.x.x"
    api_token: "YOUR_API_TOKEN"
    verify_ssl: false

default_lookup:
  "192.168.x.x": "new-device-id"

Troubleshooting

Error Cause Resolution
Connection refused Device unreachable Check network path, verify IP address, check HTTPS enabled
401 Unauthorized Invalid API token Regenerate token on FortiGate, update credentials file
403 Forbidden Insufficient permissions Check REST API admin profile permissions
SSL certificate verify failed Self-signed certificate Set verify_ssl: false in parameters
Timeout Device overloaded or slow Increase timeout, check device CPU/memory
Device not found Unknown device alias Use explicit IP address or add to device_registry
